# UI 全面改善 — 設計仕様書

日付: 2026-04-08
ステータス: 承認済み

## 目的

pbg-filemaker-migration-workbench の UI を、日本語前提・非技術者前提で全面改善する。
「誰が見ても 100% 理解できる」ことを最優先とし、英語・曖昧表現・判断に迷う状態を残さない。

## 対象ファイル

- `src/ui/public/app.js` — 全画面の HTML 生成ロジック（2699行）
- `src/ui/public/style.css` — CSS スタイル定義
- `src/ui/labels.ts` — 日本語ラベル辞書
- `src/ui/server.ts` — API エンドポイント（列レビュー保存の required フィールド変更に伴う軽微な調整）

## 非対象

- バックエンドパイプラインの仕様変更
- pbg-operations-core 側の変更
- 新機能の追加
- 英語 UI 併記

## 画面別設計

### 1. ダッシュボード（`renderDashboard`）

**現状の問題:**
- `run.mode` がそのまま英語表示される
- `mode: archive` など意味不明な表示
- 状態が「完了/失敗/実行中」の3つだけで、列確認中・設定保存済みが分からない
- 後から見返したとき何のファイルか分からない

**変更内容:**

| 項目 | Before | After |
|------|--------|-------|
| 見出し | 「直近の実行結果」 | 「最近の作業」 |
| 主表示 | `run.mode`（英語） | ファイル名（日本語） |
| mode 表示 | `mode: archive` | 非表示 |
| ファイル種別 | 非表示 | 日本語ラベルで表示（例: 顧客一覧、電話履歴） |
| 状態バッジ | 完了/失敗/実行中 | 完了/列の確認中/設定保存済み/要確認/エラー |
| 再開導線 | なし | 列確認中の場合「確認を続ける」ボタン |
| ナビゲーション | 「新規 Run」 | 「新しく読み込む」 |

**状態判定ロジック:**
- `status === 'completed'` かつ column-status で pendingCount > 0 → 「列の確認中」
- `status === 'completed'` かつ candidate profile 保存済み → 「設定保存済み」
- `status === 'completed'` → 「完了」
- `status === 'failed'` → 「エラー」
- その他 → 「処理中」

注: column-status の取得は既存の `/api/runs` レスポンスに含まれる情報で判定する。
追加 API コールが必要な場合は、ダッシュボードのパフォーマンスを考慮し、
`/api/runs` のレスポンスに `columnReviewStatus` サマリを追加する方向で検討する。
ただし、まずはクライアント側で既存情報から推定できる範囲で実装する。

### 2. 新しくファイルを読み込む（`renderNewRun`）

**現状の問題:**
- 「確認せず実行」ボタンが存在（危険）
- `encoding`、`delimiter` の英語括弧表記
- ヘッダ行チェックボックスの位置ずれ
- 実行モードが `run-all（単一ファイル全パイプライン）` など専門用語

**変更内容:**

| 項目 | Before | After |
|------|--------|-------|
| ページタイトル | 「新規 Run」 | 「新しくファイルを読み込む」 |
| 冒頭説明 | なし | 「ファイルを選んで中身を確認します。設定は通常そのままで大丈夫です。」 |
| 実行モード選択 | 6択のselect | 非表示（内部で自動判定） |
| 設定ファイル | 表示 | 非表示（デフォルト設定を使用） |
| 文字コード | `auto` / `utf8` / `cp932 selected` | 「自動検出（通常はこのまま）」がデフォルト |
| 文字コードラベル | 「文字コード (encoding)」 | 「文字コード」 |
| 区切り文字ラベル | 「区切り文字 (delimiter)」 | 「区切り文字」 |
| ヘッダ行 | `<label><input checkbox> ヘッダ行あり</label>` | 独立したカード風レイアウト + 補助説明 |
| ヘッダ行デフォルト | 常に checked | XLSX → checked、CSV → unchecked（ファイル選択時に自動切替） |
| 「確認せず実行」 | 存在する | **削除** |
| 「プレビュー」 | 「プレビュー」 | 「中身をプレビュー」 |
| ボタン配置 | 確認・実行・プレビュー3つ横並び | 「ファイルを確認する」（主）+ 「中身をプレビュー」（従） |
| ドロップゾーン | フォーム下部 | フォーム上部（最初に目に入る位置） |
| ローカルパス | 「または、ローカルパスを直接指定」 | 折りたたみ or 下部に移動（通常使わない） |

**ヘッダ行の自動切替ロジック:**
```
ファイル選択時:
  if (拡張子 === '.xlsx') → hasHeader = true
  if (拡張子 === '.csv')  → hasHeader = false
ユーザーは手動でも切替可能
```

**実行モードの扱い:**
`mode` フィールドは非表示にするが、内部では従来どおり送信する。
デフォルト値は `run-all` を維持。

### 3. 列の確認（`renderColumnReview`）

**現状の問題:**
- 1列1カード全幅で縦に長すぎる
- 「使う？」「必須？」の2つの質問が冗長で違いが分からない
- 「はい/いいえ/わからない」の3択が意味を持たない
- 入力ルール欄のラベルが「入力ルールがありますか？」で曖昧

**変更内容:**

**レイアウト:**
- PC幅（>768px）で 2カラムグリッド
- 768px以下で 1カラムにフォールバック

**列カードの構造（1カード）:**
```
┌─────────────────────────────────────┐
│ [N列目] 列名         [未回答バッジ]  │
│ 例: サンプル値1, サンプル値2         │
│                                      │
│ [何を入力する列？  ] [入力規則・選択肢] │
│ (自由記入)           (自由記入)       │
│                                      │
│ [ 使う ]  [ 使わない ]               │
│ (大きなボタン、二択)                  │
└─────────────────────────────────────┘
```

**質問の統合:**
- 「今も使いますか？」「必須ですか？」→ 削除
- 「この列をどうしますか？」→ **「使う / 使わない」二択**
- 「わからない」→ **削除**（移行にわからないは意味がない）
- 未回答 = 使わない扱い

**入力欄:**
- 「この列は何を入れる場所ですか？」→ 「何を入力する列？」に変更
- 「入力ルールがありますか？」→ 「入力規則・選択肢」に変更、右隣に配置
- 2つの入力欄は横並び（grid-template-columns: 1fr 1fr）

**視覚的フィードバック:**
- 「使う」選択時: 緑ボーダー（#16a34a）、背景薄緑
- 「使わない」選択時: グレーボーダー、カード全体をグレーアウト（opacity: 0.65）、入力欄 disabled
- 未回答: 黄色ボーダー（#fde68a）、「選ばないと使わない扱い」注記

**サマリバー（カード群の下部）:**
- 使う: N件（緑丸）
- 使わない: N件（グレー丸）
- 未回答: N件（黄丸）
- 右端に「確認を保存する」ボタン

**データモデルの変更:**
- `inUse` フィールド: `'yes' | 'no' | 'unknown'` → `'yes' | 'no'`
- `required` フィールド: 削除（UI から除去、API は後方互換のため受け取っても無視）
- `rule` フィールド: 維持（「入力規則・選択肢」として使用）
- `meaning` フィールド: 維持（「何を入力する列？」として使用）

## 既存の column-review-fields 4カラムグリッドの廃止

現行の `.column-review-fields` の `grid-template-columns: minmax(280px, 1.2fr) minmax(220px, 1fr) minmax(220px, 1fr) minmax(260px, 1.1fr)` は廃止する。
代わりに、カード内は以下の構造:

1. ヘッダー行（列番号 + 列名）
2. サンプル値
3. 入力欄2つ横並び（何を入力する列？ / 入力規則・選択肢）
4. 使う/使わない二択ボタン

## 後方互換

- `required` フィールド: API は引き続き受け取るが無視する。保存時は `'unknown'` を送信。
- `inUse` フィールド: `'unknown'` は今後生成しない。既存データの `'unknown'` は `'no'` として扱う。
- `mode` フィールド: UI から非表示にするが、内部では `'run-all'` をデフォルトで送信する。

## 受け入れ条件

1. 新規 Run 画面から「確認せず実行」が削除されている
2. ヘッダ行の表示ずれが解消されている
3. 文字コードのデフォルトが「自動検出」になっている
4. XLSX 選択時にヘッダ行が自動チェック、CSV 選択時に自動アンチェックされる
5. 列確認画面で PC 幅 2カラム表示が成立している
6. 列確認が「使う/使わない」の完全二択になっている
7. 「何を入力する列？」と「入力規則・選択肢」が横並びで表示されている
8. 使わない列のカードがグレーアウトされる
9. サマリバーで使う/使わない/未回答の件数が表示される
10. ダッシュボードで各作業のファイル名と状態が誰でも理解できる
11. 列確認結果のボタン名が目的明示になっている
12. UI 全体で英語・内部用語が大幅に減っている
13. build / lint / typecheck が通る
